# The Ploradian

The Ploradian is a zero-database, static satirical IT/business/economy newspaper. The site is generated by Astro from Markdown files in `content/articles/published`, and a Cloudflare Worker can run on Cron to generate new Markdown articles from RSS items, validate them, and commit them to GitHub.

## Structure

```txt
apps/
  web/       Astro static site
  worker/    Cloudflare Worker Cron publisher
content/
  articles/
    published/
    draft/
  sources/
    seen.json
prompts/
  satire-engine.md
```

## Local setup

```sh
npm install
npm run sync:prompt
npm run build
npm run dev:web
```

The web app is static-only. It reads Markdown from `content/articles/published/*.md`.

## Worker AI provider

The Worker defaults to OpenAI, using your `OPENAI_API_KEY` secret:

```toml
AI_PROVIDER = "openai"
OPENAI_MODEL = "gpt-5.5"
```

Cloudflare Workers AI is still available as an optional fallback:

```toml
AI_PROVIDER = "workers-ai"
WORKERS_AI_MODEL = "@cf/meta/llama-3.3-70b-instruct-fp8-fast"

[ai]
binding = "AI"
```

Only add the `[ai]` binding when switching to Workers AI. The default OpenAI path does not need it.

## Worker secrets

Do not put secrets in `wrangler.toml`. Configure them with Wrangler:

```sh
cd apps/worker
npx wrangler secret put OPENAI_API_KEY
npx wrangler secret put GITHUB_TOKEN
npx wrangler secret put CRON_SECRET
```

Non-secret defaults live in `apps/worker/wrangler.toml`:

- `AI_PROVIDER`, default `openai` in this repo
- `OPENAI_MODEL`, default `gpt-5.5`
- `GITHUB_REPO`, for example `owner/repo`
- `GITHUB_BRANCH`, default `main`
- `RSS_FEEDS_JSON`
- `MAX_ARTICLES_PER_RUN`
- `DRY_RUN`
- `SITE_TIMEZONE`

## Cron timing

Cloudflare Cron schedules are UTC. The default schedule is `0 * * * *`, which runs once per hour on the hour.
Each run publishes up to `MAX_ARTICLES_PER_RUN` articles. This deployment defaults to one article per hour and rechecks `content/sources/seen.json` immediately before committing so overlapping runs do not publish the same source twice.

## Manual run

```sh
curl -X POST https://your-worker.example.workers.dev/run \
  -H "Authorization: Bearer $CRON_SECRET"
```

Add `?dry_run=true` to avoid GitHub commits and return generated Markdown in the response.

## Local Worker dry run

Copy the local secret template and fill your OpenAI key:

```sh
cp apps/worker/.dev.vars.example apps/worker/.dev.vars
npm run dev:worker
```

Then run:

```sh
curl -X POST "http://127.0.0.1:8787/run?dry_run=true" \
  -H "Authorization: Bearer local-dev-secret"
```

## Deployment

Run the predeploy check first:

```sh
npm run predeploy
```

Items marked `USER:` are the only steps that require account-specific input, such as real secrets or the GitHub repo name.

Deploy the static web output with any static host, including Cloudflare Pages. Deploy the Worker from `apps/worker` with:

```sh
npm run deploy:web
npm run deploy:worker
```

The Worker also serves the static web build on the custom domain `news.ploradian.com`.

Generated article commits need a static asset rebuild before they appear on the live site. The repository includes a GitHub Actions workflow that runs `npm run deploy:worker` on every push to `main`. Add a repository secret named `CLOUDFLARE_API_TOKEN` with Workers deploy permission to enable automatic redeploys after Cron commits.
API endpoints stay on the same Worker:

- `GET /health`
- `POST /run`

Generated articles are committed through the GitHub REST API. The Worker never commits raw copied article bodies; it commits only generated Markdown plus `content/sources/seen.json`.
